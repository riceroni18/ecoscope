Project Documentation: Key Largo Climate Indicator Pipeline
1. Project Purpose & Ecological Objective
The goal of this geospatial data pipeline is to transition raw, large-scale biodiversity occurrence records into actionable conservation intelligence for vulnerable marine environments.

By focusing specifically on the Key Largo region within the Florida Keys National Marine Sanctuary, the project monitors a decade (2016–2026) of historical distribution data for three foundational indicator species representing distinct ecological tiers:

Red Mangrove (Rhizophora mangle): Represents coastal buffering, shoreline stabilization, and critical intertidal nursery habitats.

Stoplight Parrotfish (Sparisoma viride): Represents reef health, macroalgae grazing pressure, and coral substrate maintenance.

Great Barracuda (Sphyraena barracuda): Represents trophic apex stability and upper-level ecosystem equilibrium.

2. Technical Implementation & Architecture
The project utilizes a modular, Python-centric data science workflow to handle dense public data streaming:

[ GBIF API ] ──(Pagination Loop)──> [ Pandas Dataframe ] ──(Data Cleaning)──> [ Folium TimeSlider ]
Data Ingestion (pygbif API): Engineered an automated pipeline to query the Global Biodiversity Information Facility. To bypass strict API payload thresholds, the ingestion engine implements an offset-based pagination loop to systematically download thousands of records within a defined coordinate polygon geometry.

Data Hygiene (pandas): Programmatically sanitized raw payloads by dropping incomplete spatial vectors, removing duplicate entries, and normalizing inconsistent chronological entries into a standardized ISO format (YYYY-MM-DD).

Dynamic Temporal Visualization (folium): Converted the cleaned relational dataset into a structured GeoJSON Feature Collection. The data is rendered using Leaflet via the TimestampedGeoJson plugin, animating sightings month-by-month to evaluate range shifts over a ten-year window.

3. Advanced Data Validation & Critical Thought Process
A core focus of this project is acknowledging and evaluating the systemic limitations of open-access, crowd-sourced repositories, specifically distinguishing Sampling Bias from True Biological Trends:

The Opportunistic Sampling Limitation
An initial temporal sweep reveals an intense concentration of data points skewed heavily toward recent years (2025–2026), with sparser data in the early timeline. A naive assessment might interpret this as a sudden population explosion. However, the scientific framework of this project identifies this trend as opportunistic sampling bias driven by:

Increases in citizen-science mobile application adoption over the decade.

High human effort concentration along popular eco-tourism corridors (e.g., John Pennekamp Coral Reef State Park) vs. remote sanctuary segments.

Fair-weather and seasonal diving variations.

Future Scale Strategy: OPUE Normalization
To advance this pipeline from an observational mapping interface to an absolute population metric, a secondary iteration would scale the ingestion engine to pull all simultaneous organism metadata within the bounding box. This would allow the calculation of an Observation Per Unit Effort (OPUE) index, normalizing indicator sightings against total checklist submissions to map true demographic shifts accurately.

1. The Ingestion Layer (The while Loop)
What it does: It talks to the GBIF database using an API. Because GBIF won't let you download thousands of records all at once (it caps requests at 1,000), you wrote a loop using an offset.

How to say it: "I wrote a pagination loop that requests 1,000 records at a time, tracks how many we've downloaded, and shifts the 'offset' window forward until we've safely ingested the entire 10-year historical dataset for that polygon."

2. The Wrangling Layer (pandas)
What it does: It cleans up the messy data.

How to say it: "I used Pandas to drop any records missing coordinates or species names, stripped out exact duplicates so we didn't double-count the same observation, and forced the dates into a standardized datetime format so they could be ordered chronologically."

3. The Visual/Temporal Layer (TimestampedGeoJson)
What it does: It builds the map and the slider.

How to say it: "Standard maps get cluttered when tracking 10 years of data. To show changes over time, I converted the dataframe into a GeoJSON Feature Collection. I mapped the coordinates, assigned a custom hex color based on the species' ecological category, and attached a timestamp to each point. Then, I passed that data to Folium's time slider plugin, which dynamically controls the rendering opacity of the circle markers month-by-month."
